# MySQL setup

Backend uses MySQL for local development and deployment.

## Option A: Run MySQL with Docker

Use this when the hosted Railway database is expired or unavailable.

Prerequisite: install Docker Desktop and reopen PowerShell so `docker` is available.

```powershell
cd D:\Code\StudyHubAI\BE\aistudyhub-be
.\run-docker-db.ps1
```

The Docker database uses:

```properties
DB_URL=jdbc:mysql://127.0.0.1:3306/aistudyhub?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Ho_Chi_Minh
DB_USERNAME=root
DB_PASSWORD=123456
```

To run without changing your existing `.env`, use:

```powershell
.\run-dev-local-db.ps1
```

Alternatively, set the DB values directly in `.env`, then run:

```powershell
.\run-dev.ps1
```

Data is stored in the Docker volume `aistudyhub_mysql_data`, so it survives container restarts.
The container listens on MySQL port `3306` internally and is exposed on host port `3306`.

Useful commands:

```powershell
docker compose --env-file .env ps
docker compose --env-file .env logs -f mysql
docker compose --env-file .env down
```

Only remove the volume when you intentionally want to delete the local database:

```powershell
docker compose --env-file .env down -v
```

## Option B: Install MySQL directly

Install MySQL Server 8.x and MySQL Workbench from the official installer.

During installation, remember the root password.

### Create database

Open MySQL Workbench and run:

```sql
CREATE DATABASE IF NOT EXISTS aistudyhub
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;
```

### Run backend with MySQL

PowerShell:

```powershell
cd D:\Code\StudyHubAI\BE\aistudyhub-be
$env:DB_URL="jdbc:mysql://127.0.0.1:3306/aistudyhub?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Ho_Chi_Minh"
$env:DB_USERNAME="root"
$env:DB_PASSWORD="your_mysql_password"
$env:JWT_SECRET="change-this-to-a-long-secret-at-least-32-characters"
.\mvnw.cmd spring-boot:run
```

With `JPA_DDL_AUTO=update`, Hibernate creates/updates tables from the JPA entities.

## Current tables generated by JPA

- `users`
- `refresh_tokens`

Later modules will add:

- `categories` / `subjects`
- `documents`
- `chat_sessions`
- `chat_messages`
- OTP/reset-password tables

## Notes

- Do not commit real database passwords or JWT secrets.
- If you already have a SQL Server `.sql` file, convert table types and syntax before importing to MySQL. For the current auth module, letting JPA create the schema is safer.
